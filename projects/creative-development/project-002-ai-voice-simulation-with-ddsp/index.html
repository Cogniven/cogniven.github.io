AI Voice Simulation with DDSP

A continuing experiment in voice dataset preparation, DDSP-based training, inference testing, and iterative audio reconstruction.

Project Note

This project records several attempts to build and refine a voice simulation workflow using DDSP. The focus is not only the final model, but also the dataset decisions, preprocessing choices, inference results, failed paths, and checkpoints that shaped the process.

Latest

Update · 2026-06-18

The current training run has reached roughly 50,000 steps. This round uses audio derived from the previous training dataset after de-harmonization processing, rather than continuing with the new recording path that sounded muffled.

Several songs have been inferred with the current model. The model is already able to produce usable song outputs, but some tracks show electronic-sounding artifacts because of limitations in the original source domain.

The next plan is to choose a desired vocal source and use it for model inference. After original songs are written, this inferred model can be used as a singing voice for those works.

History

Development log

Attempt 01 Early DDSP

Initial voice-source experiment

The first attempt used an earlier DDSP workflow and a dubbing voice source. The training ran for roughly 40,000 to 50,000 steps, mainly to test whether the source material could support a usable model.

Attempt 02 DDSP-Barbara 6.1

Second dataset and longer training

The second attempt used DDSP-Barbara 6.1 and reached around 110,000 steps. This dataset later became the stronger base for the DDSP 6.3 retraining route.

2026-06-13 DDSP 6.3

New recording test and retraining decision

A new recording was processed through AU and MSST, then used for DDSP 6.3 training. The sound remained muffled, including the 6000-step inference result, so the project returned to the second-attempt dataset for retraining.

2026-06-18 ~50k Steps

De-harmonized dataset and song inference

The current model has trained for roughly 50,000 steps using de-harmonized audio derived from the previous dataset. Several songs were inferred successfully, though some outputs still carry electronic-sounding artifacts due to the original source domain.

Diagnosis

Current judgment

The reused dataset is more productive. Returning to the previous dataset after de-harmonization has produced a more usable training path than the newly recorded audio.
The model can already infer songs. The roughly 50,000-step model has produced several song inference results, which means the workflow is now useful for practical music testing.
Source-domain limits remain visible. Some songs still show electronic artifacts because the model is constrained by the domain and quality of the original source material.

Next

Prepare a voice for future songs.

The next stage is to select the desired vocal source, use it for model inference, and prepare a singing voice that can later perform original songs after they are written.
