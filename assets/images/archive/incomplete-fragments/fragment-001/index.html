The Best Shape — A QR Code Fragment

An unfinished attempt to derive a readable QR code from an ideal visual form, treating encoding rules as constraints rather than as a final output format.

Date Undated
Status Unfinished
Field Encoding / Visual Logic
Mode Fragment / Manual Search

Fragment Note

This project did not start from a domain name. It started from a harder question: could an already imagined visual form be forced backward into a valid QR code?

Intent

Starting Point

The original plan was to understand QR codes in structural terms, then use that knowledge to manually search for the most beautiful readable form. The target content did not have to be a domain name; it could also have been a company name or another short string.

Instead of generating a QR code from text, the intended direction was the reverse: first imagine an ideal composition, possibly guided by ratios such as the golden section, then test what the code could actually say while remaining readable.

Research

What Had to Be Understood First

The first layer of work was structural. A standard QR code is not a free square grid: it contains fixed function patterns such as finder patterns in three corners, separators, timing patterns, alignment patterns, format information, and in higher versions version information. These regions do not carry arbitrary message content.

The remaining encoding region is also not simply “text.” It is filled with data codewords and Reed–Solomon error-correction codewords, and then one of eight data masks is applied to avoid visually undesirable distributions. This meant that the search for form had to take place under several coupled constraints at once.

Iterations

Development Stages

Stage 01

Reading the QR Code as Structure

The first stage was analytic rather than visual. The work focused on identifying which parts of a QR code were fixed, which parts were available to encoding, how version size changes the matrix, and how format, mask, and error correction intervene in the final pattern.

Stage 02

Reverse Search for an Ideal Form

The second stage was intended to reverse the usual direction: begin from a manually designed form, then use the available regions to test what content could be encoded there while preserving both readability and aesthetic proportion. This was where the project stalled, because the calculation became too cumbersome for manual continuation.

Observations

What Became Clear

The project made clear that a QR code cannot be treated as a blank drawing field. Too much of the matrix is already occupied by functional requirements, and the remaining region is still shaped by encoding mode, codeword placement, error correction, and masking.

It also became clear that “the most beautiful QR code” is not only a visual problem. It is an inverse-encoding problem: the desired shape, the readable content, and the decoding rules must all be satisfied at the same time.

Status

Where It Stopped

The project was never completed. The planned manual search for an ideal readable form became too tedious to continue, and no final output was fixed.

What remains is the method outline: understand the QR code as a constrained structure, then attempt to reverse from form back into message. For that reason, this work is better kept as an incomplete fragment than as a finished experiment.
