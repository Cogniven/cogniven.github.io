All-Weather Portfolio China Market Parameter Optimization

2026 · Strategy Test / Excel Backtest

A practical attempt to adapt the all-weather portfolio to the Chinese market, moving from strategy scripting to manual Excel backtesting and parameter tuning.

Spreadsheet
Open the Excel backtest file

Overview

This project started from the idea of applying the all-weather portfolio to the Chinese market. The purpose was not simply to imitate the classic foreign allocation model, but to test whether a simplified domestic version could be made workable under a clearer and more controlled asset structure.

In practical terms, the portfolio was reduced to four components: CSI 300, gold, treasury bond ETF, and cash. These corresponded to equity exposure, inflation-sensitive asset, defensive bond allocation, and liquidity reserve.

Method

Historical data for the four assets was collected and aligned into a unified sequence. The original plan was to write a formal strategy file and validate the allocation and rebalancing logic using a standard backtesting method. That attempt failed to produce a usable result, so the project could not continue through its intended automated route.

After that, the entire model was rebuilt in Excel. The portfolio process was reconstructed directly with formulas, including price updates, position value, weight changes, rebalancing signals, turnover, and fee effects. In this version, weekend gaps were handled by carrying forward Friday prices, and trades were assumed to take place at 8:00 every morning.

The rebalancing logic began from equal weights. Each asset started at one quarter of total capital. A parameter k was introduced as the rebalancing threshold: once any asset weight moved above or below 25% ± k, the portfolio would rebalance back to equal allocation.

Data Range

The current dataset used in the Excel model runs from 2013-07-29 to 2025-12-09. This period covers a relatively long cross-cycle interval and provides enough span to observe the interaction between allocation, rebalancing threshold, and long-term portfolio stability.

Excel Framework

The workbook already contains price data, parameter inputs, backtest calculations, chart output, and parameter adjustment sections. In other words, Excel is not only a record of results here, but the actual environment in which the strategy was rebuilt and tested after the scripted route failed.

This manual framework makes each part of the process explicit: asset values, total equity, asset weights, rebalance signals, turnover, fees, historical peak, drawdown, and parameter comparison can all be traced directly in the file.

Result

The final outcome was not a clean system-level strategy implementation, but a transparent manual backtesting framework. What this project shows is not only a domestic adaptation of the all-weather idea, but also a change in method: when the intended technical path failed, the strategy was rebuilt as an explicit calculation structure.

The result is slower and more manual, but also easier to inspect. Every assumption is visible, every parameter can be adjusted, and the logic of the portfolio remains open rather than hidden inside a failed automated process.
